# Production Plan

## Phase 1 - Scaffold
Create the repo brain, Godot project, first scene, truck prototype, and a minimal delivery loop.

## Phase 2 - Core Prototype
Prove the main loop:
- drive
- pick a contract
- manage cargo
- deliver
- see a city reaction
- get a shift summary

## Phase 3 - Vertical Slice
Add multiple districts, more cargo behavior, more event pressure, upgrades, and a stronger visual/audio identity.

## Phase 4 - First Playable Demo
Polish the core loop, increase content variety, and make the game streamable and repeatable.

## Phase 5 - Alpha
Expand district content, cargo families, faction interactions, route memory, and rare incidents.

## Phase 6 - Beta
Balance, optimize, improve UI clarity, and harden the experience.

## Build Order
1. Truck controller
2. Camera
3. Route selection
4. Cargo state system
5. Delivery objective system
6. District mood and reputation
7. Dispatcher messaging
8. Shift summary
9. Event system
10. Upgrades
11. Rare incidents
12. World expansion
13. Audio and polish
